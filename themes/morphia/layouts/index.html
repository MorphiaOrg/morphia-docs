{{/* ── Concept / Pull-quote ── */}}
Treat your data the way MongoDB does — as documents.
Treat your code the way Java does — with types.
  1. i.

    Bidirectional. Round-trip every entity. Whatever goes into the collection comes back as the same shape, the same types, the same indexes.

  2. ii.

    Type-safe. Operators only appear where they make sense. Compile-time checks for queries that used to break at 3 a.m.

  3. iii.

    Native. Aggregation, geospatial, change streams, transactions — all of MongoDB, none of the boilerplate.

{{/* ── Features ── */}}
features

Everything you need to ship document-first Java — without the glue.

01

Bidirectional mapping

Annotate POJOs, persist them. Round-trip to BSON and back without writing a single converter.

@Entity · @Id · @Property
02

Type-safe queries

Build queries with operators that only show up where they make sense. No more typo-driven outages.

find(Post.class).filter(eq(...))
03

Aggregation framework

Full pipeline support — match, group, lookup, project, unwind — typed against your entities.

$match · $group · $lookup
04

Annotation-driven indexing

Declare indexes alongside the fields they cover. Compound, sparse, unique — all in source.

@Indexed · @CompoundIndex
05

Geospatial queries

2dsphere indexes, $geoNear, $geoWithin — first-class support for Point, Polygon, and friends.

@GeoIndex · Point · Polygon
06

Custom type converters

Bring your own Money, Email, JSR-310 types. Implement Codec once and forget about it.

Codec<MyType>

in practice

Read like Java. Run like Mongo.

QueryExample.java type-safe queries 
var ds = Morphia.createDatastore(client, "blog");

Query<Post> q = ds.find(Post.class)
  .filter(eq("author", ada))
  .filter(gte("score", 0.8))
  .filter(in("tags", "java", "mongo"));

List<Post> top = q
  .iterator(new FindOptions()
    .sort(descending("score"))
    .limit(10))
  .toList();
Aggregation.java native pipelines 
ds.aggregate(Post.class)
  .match(eq("published", true))
  .group(id("$author"),
    sum("posts", 1),
    avg("score", "$score"))
  .sort(descending("posts"))
  .limit(5)
  .execute(AuthorStats.class)
  .toList();
