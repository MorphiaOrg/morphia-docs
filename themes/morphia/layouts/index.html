Why Morphia?

Bidirectional Mapping

Transparently serialize Java entities to MongoDB documents and back. Annotate once, persist anywhere.

Type-Safe Queries

Build queries with a fluent Java API that mirrors MongoDB's query operators, with full type safety and IDE completion.

Full Aggregation Support

Use MongoDB's aggregation framework directly from your Java classes — no raw BSON wrangling required.

Annotation-Driven Indexing

Declare compound, text, and geospatial indexes alongside your model with @Indexes and @Index.

Extensible Type Conversion

Register custom codecs for any type. Morphia's codec pipeline slots cleanly into the MongoDB Java driver.

Geospatial Queries

Query by proximity, within polygons, or along routes with first-class support for GeoJSON and legacy coordinate pairs.

Less plumbing, more domain logic

Map a POJO with a single annotation, then save, query, and update using a high-level API that stays out of your way.

Mapping
@Entity("books")
@Indexes(@Index(fields = @Field("author")))
public class Book {
    @Id private ObjectId id;
    private String title;
    private String author;
}
Querying
Datastore ds = Morphia.createDatastore(client, "library");

List<Book> results = ds.find(Book.class)
    .filter(eq("author", "J. Bloch"))
    .iterator()
    .toList();
